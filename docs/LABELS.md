# Repository Label System Specification

This document details the standardized label taxonomy used across the **CineVerse** GitHub repository to organize issues, pull requests, and project boards.

---

## 🏷️ Primary Category Labels

| Label Name | Badge Color | Hex Code | Description |
| ---------- | ----------- | -------- | ----------- |
| `good first issue` | 🟣 Purple | `#7057ff` | Beginner-friendly task suitable for first-time contributors. |
| `help wanted` | 🟢 Teal | `#008672` | Community assistance desired; open for assignment. |
| `documentation` | 🔵 Blue | `#0075ca` | Guide, API ref, or code comment enhancements. |
| `bug` | 🔴 Red | `#d73a4a` | System malfunction or unintended behavior. |
| `enhancement` | 🩵 Light Blue | `#a2eeef` | Feature additions or behavioral upgrades. |
| `security` | 🚨 Bright Red | `#ee0701` | Authentication, authorization, or vulnerability patches. |
| `performance` | 🟪 Light Violet | `#d4c5f9` | Latency reduction, caching, or bundle size fixes. |

## 🚥 Priority Labels

- `high priority` (`#b60205`): Urgent bugs or blocking release items.
- `low priority` (`#cfd3d7`): Non-critical improvements and minor cosmetic tweaks.
